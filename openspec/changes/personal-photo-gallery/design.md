## Context

这是一个面向个人用户的照片图库系统，目标是提供简单、快速、易用的个人照片管理体验。系统不需要支持多租户，不需要复杂照片处理，仅需实现上传、浏览、相册管理和基础身份验证。

## Goals / Non-Goals

**Goals:**
- 提供清晰的个人照片浏览和管理界面
- 实现可靠的照片上传与存储
- 支持相册和标签，用于个人照片整理
- 使用简单身份验证保护个人图库

**Non-Goals:**
- 不实现多租户或团队权限模型
- 不实现照片压缩、缩略图生成、AI 标签、审核或分享逻辑
- 不实现复杂的外部分享或协作功能

## Decisions

- 架构上采用单一应用或小型服务组合，以保持简单可用。后端可用一个服务处理认证、照片元数据和存储接口，避免不必要的微服务复杂度。
- 照片文件存储使用对象存储（如 S3/MinIO），元数据保存在关系型数据库（如 PostgreSQL/MySQL）。这样可分离文件与结构化信息，同时保留简单可用性。
- 身份验证采用基础登录会话机制，满足个人用户的访问控制需求，无需 OAuth 或复杂用户管理。
- 前端采用单页应用模式，提供照片网格、详情和管理操作，界面简洁直观，避免过多配置选项。
- 不在当前阶段实现异步图片处理流程，上传即保存，后续可按需扩展为异步处理。

## Risks / Trade-offs

- [风险] 个人图库功能扩展后，单一服务可能需要拆分。 → 通过清晰模块边界设计，后续可拆成认证、照片元数据、存储三个服务。
- [风险] 当前不做缩略图生成可能导致列表页首次加载较慢。 → 选择合理默认展示尺寸并限制一次加载数量。
- [风险] 简单登录机制可能不足以应对高级安全需求。 → 明确该项目仅面向个人使用，必要时可后续加强认证策略。
